# Travel Planner Assistant - Frontend

A React-based web application that provides an interactive chat interface for the AI Agent Travel Planner. Users can submit travel requests and receive personalized travel plans generated by specialized AI agents coordinated through Azure Durable Functions.

## Features

- **Travel Request Form**: Collect user preferences including destination preferences, budget, duration, and special requirements
- **Real-time Progress Tracking**: Visual progress indicator showing the orchestration status as AI agents work on your plan
- **Interactive Chat Interface**: View travel plans and agent responses in a conversational format with Markdown rendering
- **Human-in-the-Loop Approval**: Review and approve/reject generated travel plans before booking
- **Responsive Design**: Works across desktop and mobile devices

## Tech Stack

- **React 19** - UI framework
- **Vite 7** - Build tool and development server
- **Axios** - HTTP client for API communication
- **React Markdown** - Render Markdown content in chat messages
- **Azure Static Web Apps** - Deployment target

## Available Scripts

### `npm start` or `npm run dev`

Runs the app in development mode at [http://localhost:3000](http://localhost:3000).

The page will hot-reload when you make changes.

### `npm run build`

Builds the app for production to the `build` folder. The build is minified and optimized for best performance.

### `npm run preview`

Locally preview the production build.

## Environment Variables

Create a `.env` file for local development or `.env.production` for production builds:

```env
VITE_API_URL=http://localhost:7071/api
```

| Variable | Description | Default |
|----------|-------------|---------|
| `VITE_API_URL` | Backend API endpoint URL | `http://localhost:7071/api` |

## Local Development

1. **Install dependencies**:

   ```bash
   npm install
   ```

2. **Set up environment variables**:

   Create a `.env` file in the Frontend directory:

   ```env
   VITE_API_URL=http://localhost:7071/api
   ```

3. **Start the development server**:

   ```bash
   npm start
   ```

4. **Ensure the backend is running**:

   The frontend expects the Azure Functions backend to be running at the configured `VITE_API_URL`.

## Deployment

This application is designed to be deployed to Azure Static Web Apps. See the main project README for deployment instructions using Azure Developer CLI (`azd`).

### Manual Build for Deployment

```bash
# Build for production
npm run build

# The build output will be in the 'build' directory
```

## API Integration

The frontend communicates with the following backend endpoints:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/travel-planner` | POST | Submit a new travel request |
| `/api/travel-planner/status/{instanceId}` | GET | Check orchestration status |
| `/api/travel-planner/approve/{instanceId}` | POST | Approve or reject a travel plan |
| `/api/travel-planner/confirmation/{instanceId}` | GET | Check booking confirmation status |
